# 多人音游推荐歌曲计算器

本文件用于给智能体/协作者提供**页面主题与 UI/UX 约束提示词**，目标是让界面始终保持**简易、清晰、低负担**，并与当前主页（`src/pages/index/index.vue`）的极简卡片风格一致。

## 页面主题（必须遵守）

- **关键词**：极简、清爽、信息密度适中、可读性优先、减少装饰、减少认知负担
- **整体观感**：浅色/深色都要成立；灰阶为主 + 少量强调色；圆角卡片/面板；清晰的分组与标题层级
- **交互策略**：默认可用；即时反馈（loading / error / empty）；不做花哨动效

## 视觉规范（建议作为默认设计 token）

- **圆角**：卡片/面板使用 12px；输入/按钮使用 10px；胶囊标签使用 999px
- **间距**：以 10/12/14/16/24 为主（与现有页面一致）；块与块之间留白清楚
- **边框**：1px 细边框（`#e6e6e6` / `#eee` / `#ddd`）做层级；避免重阴影
- **颜色**：
  - 浅色（Light）：
    - 主背景：`#fff`；次背景：`#fafafa`
    - 主文本：`#111`；次级文本（muted）：`#666`；更弱信息：`#888`
    - 强调色：允许 1 个低饱和强调色（如蓝/青），仅用于按钮主操作、关键状态提示
  - 深色（Dark / 黑夜模式）：
    - **不要纯黑**（避免刺眼与层级丢失），推荐深灰蓝底：主背景如 `#0f1115`，次背景如 `#131723`
    - 面板/卡片使用略亮一档的“表面色”，例如 `#151a23` / `#101520`，并用更深的边框分层（如 `#2a3140`）
    - 主文本为偏灰白（如 `#e9edf5`），次级文本不要太暗（如 `#aeb6c5` / `#8f97a7`），确保对比度与可读性
    - 强调色同样保持低饱和（如 `#8ab4ff`），避免荧光色
- **字体**：系统字体栈；正文 13–14px；标题 16px/28px（小标题/主标题）

## 黑夜模式实现约束（必须遵守）

- **统一 token**：颜色必须走 CSS 变量/Token（例如 `--c-bg/--c-text/--c-border/--c-primary`），禁止在页面里到处写死 `#fff/#111`。
- **系统跟随 + 手动切换**：默认跟随系统（`prefers-color-scheme`），允许用户手动选择浅色/深色并持久化（localStorage）。
- **表单控件一致**：设置 `color-scheme` 并保证 input/select/button 在深色下仍清晰（背景、边框、focus）。
- **可访问性**：深色下的 muted 文本不能过暗；focus ring 必须清楚；禁用态仍可辨识。

## 布局与组件（推荐实现方式）

- **页面结构**：header（标题+一句说明）→ 多个 panel（面板）→ panel 内部 card / list
- **信息分组**：每个面板只回答一个问题（来源 / 玩家设置 / 结果），避免“巨型表单”
- **表单排版**：标签 + 控件对齐，优先两列（桌面端），窄屏自动变单列
- **列表呈现**：结果列表用卡片项 + 关键元信息 + 标签（pill）；限制渲染数量避免卡顿

## 文案风格（必须遵守）

- **短句**：一句话说清楚，不堆术语；必要术语给中文解释（例如“定数/难度”）
- **状态文案**：必须覆盖并易读：加载中、请求失败（是否回退缓存）、暂无结果、展示上限提示
- **一致性**：同一概念命名统一（玩家 1/2、期望定数、可接受区间、排序优先级）

## 可用性与可访问性（必须遵守）

- **对比度**：文本与背景对比足够；灰文字不要太浅
- **可点击性**：按钮/选择器/输入框的 hover/focus 清晰；禁用态可辨认
- **键盘可用**：表单控件可 Tab 导航；避免把关键交互只放在鼠标事件里
- **响应式**：窄屏下保持一列、间距不挤压、信息不溢出（如 meta 的 nowrap 要谨慎）

## 生成/改版 UI 的提示词（直接可复制给智能体）

> 下面每条都可以当作“系统/项目提示词”叠加使用；默认主题是**简易清晰**。

- **基础风格**：请使用浅色极简风格（并同时提供黑夜模式），卡片/面板分组清晰，灰阶为主，少量低饱和强调色；不要渐变/霓虹/拟物/重阴影。
- **排版**：标题层级明确（h1/h2/h3），正文行高舒适；表单标签与输入对齐，窄屏自动变单列。
- **组件**：按钮、输入框、下拉框统一圆角与边框；列表项用卡片承载；结果信息用 pill 标签展示。
- **状态**：所有异步与筛选都要有 loading/error/empty 文案；错误提示要说明是否使用缓存回退。
- **性能**：结果列表限制渲染数量或虚拟列表；避免在模板里做重计算。
